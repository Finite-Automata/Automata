What is Finite Automata?

Finite automata (FA) is a simple idealized machine used to recognize patterns within input. The job of an FA is to accept or reject an input depending on whether the pattern defined by the FA occurs in the input.

Explanation:


Finate Automata has the idea of:


  • A system changing state due to inputs supplied to it.
  • Some of these states being acceptor or final states.

In general, the key definition of Finite Automata is a 5-tuple (Q,Σ,I,F,δ) where:

  • Q = the finite set of States.
  • ∑ = the finite set of Inputs.
  • I = the Initial/Starting State.
  • F = finite set of Final/Accepting States.
  • δ = Transition Function.

Finite automata takes the string of input and changes its state accordingly. When the desired input is found, then the transition occurs. At the time of transition, the automata can either move to the next state or stay in the same state.



Finite Automata can be classified into two types:


Deterministic Finite Automata

Deterministic Finite Automata (DFA) - from the word deterministic, this means that it can only be in, and transition to, "one state" at a time for a given input. A DFA can have a transition (change of state) based on the set of inputs it has.

The transition function of a DFA can be defined as δ: Q x ∑→Q. This means that the cross product of an element of set Q(states) and an element of set ∑(inputs) will have a transition to the element of set Q.

Graphical Representation of DFA

  1. The states are represented by vertices (circles).
  2. The arc or arrow labeled with input/s show the transitions.
  3. The initial state is pointed by plain arrow.
  4. The final state/s are marked with a double circle.

Non-Deterministic Finite Automata

Non-Deterministic Finite Automata (NFA) - from the word non-deterministic, has a principle where it can be in, and transition to, "multiple states" at once depending on the given input. In NFA, there might be several possible final states that could result from reading a given string of inputs; as long as at least one of them is a final state, then the string is accepted.

Now that we know a NFA may transition to multiple state at a time, then it is obvious that the transition function of a NFA is different from DFA which can be defined as δ: Q x ∑→ 2 ^ Q. This means that the cross product of an element of set Q(states) and an element of set ∑(inputs) will have a transition to the element of power set of Q, not just from the elements of set Q itself.

Power set means "the set of all subsets of a given set", for example we have Q = {A,B,C}, the Power set of Q or P(Q) = {{∅},{A},{B},{C},{A,B},{A,C},{B,C},{A,B,C}}.

Graphical Representation of NFA

  1. The states are represented by vertices (circles).
  2. The arc or arrow labeled with input/s show the transitions.
  3. The initial state is marked with plain arrow.
  4. The final state/s are marked by double circle.
